Setting up your environment

Welcome to Pokerwars! In this tutorial you will learn how to build a simple bot using Javascript. You can switch your programming language anytime with the dropdown selector at the top right. Let's get started!

The starter projects

In our GitHub repository you can find starter projects for a number of langauges. You are not limited to those, you can use any programming language you want as long as it's capable of sending and receiving HTTP requests.

For Javascript, our starter project is here. We will build that same starter bot now step by step. At the end, you will get an understanding of the game and the platform, and a working bot ready for you to implement your own unique strategy. How cool is that!

Express

In this example we will be using Express, the most popular web framework for Node.js, but you can use any other similar alternative. We chose Express because it comes with a myriad of HTTP utility methods and middleware, making it super quick and easy to use.

We start by requiring and initialising express at the top. Additionally we set up a property port where we will store the port where the application will be listening. Note how we set this in a way that we can use an environment variable to override the default port 8090. Finally, we use body-parser middleware to parse incoming requests as JSON. The "extended" syntax allows for rich objects and arrays to be encoded into the URL-encoded format, allowing for a JSON-like experience with URL-encoded. More information about body-parser can be found here.

For more information about Express check out their website here.

Routes and functions

Next we need to declare some routes. These are endpoints that our application will be exposing when it's up and running, and that will be called by the Pokerwars Engine! This Engine is where all the magic happens. It keeps track of all the tournaments and calls each bot for the next action. For now, let's not worry about that and let's focus on how to create routes.

In Express, we can create a new route using app.route(). In the code, we went ahead and created a route for the play action for you. This is the endpoint the Engine will call (with a POST request) during the tournament to ask you for your next action when your bot is playing. The endpoint itself needs to follow a convention where we have /pokerwars.io/ followed by the name of something the Engine undertands, in this case play. Once that endpoint is called, our applcation will call a function named play() passing the HTTP request and HTTP response objects.

var play = function (req, res) {};

For now we'll leave this play function empty. In the next steps we'll see how it works.

In addition to play there are another two endpoints the game requires you to implement:

  • ping: used by the Engine when your bot subscribes to verify that is alive and responding.
  • notifications: this one is optional, you decide if you want them when you subscribe. Your bot does not need to necessarily provide an endpoint for these, but they can give you useful hints on what is happening during the tournaments. Please do not use notifications in your bot logic, as they are not guaranteed to always be sent. They are only informative so you are aware of what your bot is doing.

Go ahead and create similar routes and functions for ping and notifications. The ping route needs to be GET, and the notifications route needs to be POST.

HINT: create ping first and notifications second for the validation to pass.
